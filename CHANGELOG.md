# Changelog

이 프로젝트의 주요 변경사항을 기록한다. 포맷은 [Keep a Changelog](https://keepachangelog.com/en/1.0.0/)를 따른다.

버전 체계는 `YY.M.PATCH`이며 릴리즈마다 PATCH를 1씩 올린다. manifest `version`, git 태그, GitHub 릴리즈가 항상 동일해야 한다. 자세한 규칙은 [CLAUDE.md](CLAUDE.md) 참고.

## [26.6.18] - 2026-06-26

### Added
- **이모지 렌더** — 모노크롬 Noto Emoji 폰트(OFL) 번들 + 문자별 폰트 폴백. 이모지가 두부(□) 대신 텍스트 색 단색으로 표시.
- **MTU** 기기 페이지 number 엔티티 (구성 대화상자에 더해 바로 조절).
- 상태표시 엔티티 **드래그앤드롭 순서 변경** (구성 → 표시할 엔티티, EntitySelector reorder).

### Changed
- 속도 / 정지 / MTU number 엔티티를 **슬라이더**로.
- 상태표시 행에서 **기기명 접두사 제거** → `{장소} {엔티티명} {값}` (예: "거실 공기청정기 공기질 22" → "거실 공기질 22").

## [26.6.17] - 2026-06-26

### Added
- **MTU 재정의** 옵션 (구성 → MTU). HA가 보고하는 MTU가 23으로 작아 전송이 ~1바이트 청크로 쪼개져 느릴 때, 수동으로 더 큰 값(예: 247·512)을 지정하면 청크 수가 급감해 빨라진다. `0` = 자동(협상값 사용). 주의: 실제 협상 MTU보다 크게 잡으면 쓰기가 실패할 수 있으니, 표시가 깨지면 값을 낮춘다.

## [26.6.16] - 2026-06-26

### Added
- 제어/전송마다 INFO 로그 — 전원·밝기·raw 동작, 그리고 각 표시 전송의 `mtu`·청크 수·소요 시간. 연결 시 협상된 MTU도 기록.
- 협상 MTU가 23 이하이면 경고 — 전송이 ~1바이트 청크로 쪼개져 매우 느려짐(ESPHome BT 프록시 구버전 등). 임시 회피: 색 타입을 `mono`로.

### Fixed
- 연결 끊김 콜백에서 죽은 클라이언트 참조를 비우지 않아, 다음 전송이 끊긴 클라이언트를 재사용하려다 매번 재연결하던 문제(`_on_disconnect`에서 client 정리).

## [26.6.15] - 2026-06-26

### Added
- **좌우 반전 / 상하 반전** 스위치 — 패널이 뒤집혀 설치된 경우 출력을 미러링. 텍스트·이미지·GIF·단색·상태표시 모든 표시에 적용.

### Fixed
- 효과(effect)가 GIF·단색·상태표시에 적용되지 않던 문제 — 멀티프레임 경로(`legacy_gif_source`)가 effects 바이트를 0으로 하드코딩하고 있었음. 이제 설정한 효과(좌/우/상/하 등)가 전송되고, 상태표시 항목 전환에도 효과가 적용된다.

## [26.6.14] - 2026-06-26

### Fixed
- 상태표시 ON 상태에서 갱신 주기·효과·속도·정지를 바꿔도 다음 주기(최대 600초)까지 반영되지 않던 문제 — `async_track_time_interval`이 주기 경과 후에야 처음 발화하기 때문. 이제 (재)스케줄 시 즉시 1회 전송 후 주기 시작 → 설정 변경이 바로 반영된다.

## [26.6.13] - 2026-06-26

### Changed
- 상태표시가 설정된 **모든 엔티티를 한 번에 멀티프레임**으로 전송 — 기기가 GIF 프레임처럼 순차 재생(프레임당 정지 시간 = 정지 설정). 매 틱 한 항목씩 다시 보내던 회전 방식 제거. BLE 전송 1회로 끝나고 순환은 기기가 담당.

### Added
- `examples/dashboard.yaml` — 단색/이미지/GIF 버튼 + 설정 엔티티 카드 예시.

## [26.6.12] - 2026-06-26

### Added
- 기기 페이지에 **효과**(select), **속도**·**정지**(number) 설정 엔티티 추가. `텍스트 표시`(Display text) 박스와 상태표시가 이 값을 사용 — 개발자 도구 없이 글자만 입력해도 설정한 효과/속도/정지로 스크롤된다. 값은 옵션에 영속.

## [26.6.11] - 2026-06-26

### Added
- 한국어 번역(`translations/ko.json`) — `상태표시`, `갱신 주기`, `텍스트 표시`, 서비스/옵션 한글화.
- 브랜드 아이콘(글래스 LED 매트릭스) — home-assistant/brands 제출용 `icon.png`(256), `icon@2x.png`(512).

### Changed
- 상태표시 행 포맷을 `{장소} {엔티티 이름} {값}`으로 변경 (area 레지스트리에서 장소 조회, 없으면 생략).

## [26.6.10] - 2026-06-26

### Added
- **Pretendard 폰트 번들** (`custom_components/iledcolor/fonts/`, OFL) — HA(리눅스 컨테이너)처럼 시스템 CJK 폰트가 없는 환경에서도 한글이 렌더된다.

### Changed
- 폰트 로드를 `lru_cache`로 캐싱 — 매 렌더마다 폰트를 11회씩 재로드하던 것을 1회로. 텍스트/상태표시 갱신 속도가 크게 개선된다.

## [26.6.9] - 2026-06-26

GIF/텍스트/이미지 효과·전송 개선 + 개발용 GUI.

### Added
- `display_color` 서비스 — 패널 단색 채우기.
- 이미지/GIF **파일 업로드** 지원 (`file_upload` 의존성, 서비스 다이얼로그에서 직접 업로드).
- `display_image`에 배경 키아웃(`background`/`tolerance`)·`effect`·`speed`·`dwell` 추가.
- `display_gif`에 `effect`·`speed` 추가.
- `display_text`에 `dwell`(정지시간) 추가.
- effect 코드 정리 — 0 정지 / 1 좌 / 2 우 / 3 상 / 4 하 / 5 눈송이 / 6 두루마리 / 7 레이저.
- 개발용 직접통신 도구 — CLI `iledcolor_display.py`, Tkinter GUI `iledcolor_gui.py`(자동 스캔·미리보기·effect 애니메이션·버튼 상태 관리).

### Changed
- 벌크 전송을 윈도우 스트리밍(in-flight 32)으로 변경 — 작은 전송은 빠르고 큰 전송은 안정적.
- legacy GIF 포맷 수정 — 프레임 뒤에 trailing `be16(speed)` 부착, stayTime(`frame_hold`, 기본 10)을 speed와 분리. 단일 프레임 GIF도 동일 포맷.
- 투명 알파를 검정으로 합성, 한글 등 CJK 폰트 자동 선택.
- 텍스트/이미지 스크롤 시 `dwell=0`으로 무정지 연속 가능.

### Fixed
- GIF 컬러 깨짐 — multi-frame pixel data에 trailing speed 누락이 원인.
- GIF 배경 흰색 노출 — `background` 키아웃으로 해결.
