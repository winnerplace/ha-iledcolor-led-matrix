# Changelog

이 프로젝트의 주요 변경사항을 기록한다. 포맷은 [Keep a Changelog](https://keepachangelog.com/en/1.0.0/)를 따른다.

버전 체계는 `YY.M.PATCH`이며 릴리즈마다 PATCH를 1씩 올린다. manifest `version`, git 태그, GitHub 릴리즈가 항상 동일해야 한다. 자세한 규칙은 [CLAUDE.md](CLAUDE.md) 참고.

## [26.6.26] - 2026-06-26

### Fixed
- **상태표시 값이 영문으로 나오던 문제** — season(`spring`)·달 상태(`full_moon`) 등 열거형 센서가 HA 화면에선 한글인데 LED에는 원본 영문으로 표시됐다. 엔티티 이름은 이미 번역돼 나왔지만 상태값은 `state.state` 원본을 그대로 썼던 탓. HA 프론트와 동일한 순서(등록부 `translation_key`+platform → `device_class` → 기본)로 번역 카탈로그를 조회해 화면과 같은 한글로 표시하도록 수정. 번역이 없는 센서는 기존대로 원본값을 유지한다.

## [26.6.25] - 2026-06-26

### Changed
- **슬라이드를 벤더 앱과 동일한 네이티브 스크롤로 재구현** — 앱 소스(`TextSourceBean`) 분석 결과, 스크롤은 텍스트를 패널 폭짜리 **연속 프레임 `ceil(글자폭/패널폭)`장**(예 3~4장)으로 잘라 effect=좌(1) 멀티프레임으로 보내면 기기가 그 사이를 부드럽게 스크롤하는 방식. 26.6.24의 픽셀 단위 소프트웨어 스크롤(최대 120프레임)을 이 방식으로 교체 → 프레임 수 급감, 전송이 훨씬 빠름. 상태표시 멀티프레임 경로와 동일해 검증된 동작.

### Added
- **글자 크기(슬라이드)** number 엔티티 — 슬라이드 시 글자 높이를 고정(기본 **12**, 앱 기본값과 동일, 6~32). 96×16 패널에서 12면 화면당 약 8글자.

## [26.6.24] - 2026-06-26

### Fixed
- **슬라이드 글자 깨짐·2중 표시 수정** — 펌웨어가 패널보다 넓은 source를 스크롤하지 못해(framebuffer 폭 = 패널 폭) 픽셀이 행 경계에서 어긋나 깨지고 2중으로 보이던 문제. 슬라이드를 **소프트웨어 스크롤**로 변경: 큰 글자로 넓게 렌더한 뒤 패널 폭 창으로 잘라 움직이는 프레임(GIF)으로 전송. 프레임 폭이 항상 패널 폭이라 깨짐 없음. 글자 끝까지 흐른 뒤 처음으로 순환.

### Note
- 슬라이드는 긴 글일수록 프레임이 많아져(최대 ~120) 풀컬러 패널에선 전송이 다소 느릴 수 있다. 모노 패널은 빠름.

## [26.6.23] - 2026-06-26

### Added
- **컬러 이모지** — `freetype-py`로 COLR/CPAL·CBDT·sbix 컬러 글리프를 래스터화. 선택 폰트(Mona12 등)에 컬러 이모지가 있으면 그대로 쓰고, 없으면 번들 Mona12의 컬러 이모지로 폴백. Pillow는 COLR을 못 그려서 우회한 것. 풀컬러 패널에서 의미 있음. `freetype-py` 미탑재 환경에선 기존 모노 이모지로 자동 폴백.
- **슬라이드(고정 크기) 표시** — 텍스트가 길어도 글자 크기를 줄이지 않고 고정한 뒤 가로로 흘림(패널보다 넓게 렌더 → 자동 좌 스크롤). `슬라이드` 스위치 + `display_text` 서비스 `slide` 파라미터. 끄면 기존처럼 한 화면에 맞춰 축소.

### Changed
- 의존성 `freetype-py>=2.4.0` 추가.

## [26.6.22] - 2026-06-26

### Changed
- 전원이 꺼져 있으면 **어떤 콘텐츠도 전송하지 않음** — 텍스트를 바꿔도 패널을 자동으로 켜지 않는다(26.6.21의 전송 시 자동 점등 제거). 변경 후 **수동으로 켜면** 그때 표시: 텍스트 입력 모드는 마지막 입력 텍스트, 상태표시 모드는 상태 화면을 재생.

## [26.6.21] - 2026-06-26

### Added
- **표시 모드** select(제어 항목) — 상태표시 ↔ 텍스트 입력을 한 곳에서 전환. 텍스트 입력 박스에 글을 넣으면 자동으로 텍스트 모드로 바뀌어 상태표시 회전이 덮어쓰지 않는다.

### Changed
- **전원이 꺼져 있으면 상태표시 갱신을 보내지 않음.** 텍스트 등 콘텐츠를 전송하면 패널을 자동으로 켠다. 전원을 켜면 상태표시 모드일 때 즉시 1회 갱신.
- 상태표시 on/off 스위치(구성)를 제거하고 **표시 모드 select**로 대체 — 상태표시/텍스트 입력 모드 혼동 해소. 기존 설정은 자동 이행(켜짐 → 상태표시, 꺼짐 → 텍스트 입력).
- 예시 대시보드 제어 순서를 **전원 > 텍스트 > 색상**으로 정리. (기기 페이지 자체는 이름 알파벳순 고정이라 통합 코드로 순서를 못 바꾼다 — 순서가 필요하면 대시보드로.)
- 패널 전원 라이트 `is_on`을 기기 전원 플래그에 연동 — 전송 시 자동 점등이 라이트 상태에 즉시 반영.

## [26.6.20] - 2026-06-26

### Added
- **텍스트 색상** RGB 라이트 엔티티 — HA 색상 피커로 글자색 지정. ON = 고정색 사용, OFF = 기본 흰색.
- **랜덤 색상** 스위치 — 켜면 갱신마다 상태표시 행/프레임별로 선명한 랜덤색(HSV 채도·명도 최대). 고정색보다 우선.
- 기기 페이지 설정 엔티티: **패널 너비 / 패널 높이 / 글자 굵기**(number), **폰트 / 색 타입 / 프레임 세대**(select).

### Changed
- 패널 너비·높이·MTU·폰트·굵기·색 타입·프레임 세대를 **구성(옵션) 대화상자에서 기기 설정 엔티티로 이동**. 옵션 대화상자에는 표시할 엔티티 선택만 남김. 기존 옵션 값은 같은 키를 그대로 읽어 마이그레이션 불필요.

### Fixed
- 패널 전원 라이트가 HA 재시작 후 실제 상태와 무관하게 항상 꺼짐으로 표시되던 문제 — `RestoreEntity`로 마지막 ON/OFF·밝기를 복원.

## [26.6.17] - 2026-06-26

### Added
- **MTU 재정의** 옵션 (구성 → MTU). HA가 보고하는 MTU가 23으로 작아 전송이 ~1바이트 청크로 쪼개져 느릴 때, 수동으로 더 큰 값(예: 247·512)을 지정하면 청크 수가 급감해 빨라진다. `0` = 자동(협상값 사용). 주의: 실제 협상 MTU보다 크게 잡으면 쓰기가 실패할 수 있으니, 표시가 깨지면 값을 낮춘다.
